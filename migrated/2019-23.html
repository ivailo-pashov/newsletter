CQRS and Event Sourcing Intro for Developers

Command-Query Responsibility Segregation is a pattern that tells us to separate operations that mutate data from the ones that query it. It is derived from Command Query Separation (CQS), which states that there can be only two kinds of methods in a class: the ones that mutate state and return void and the ones that return state but do not change it.
If you want to know what does CQRS enable, is CQRS an Architecture and an example of the implementation, make sure to check out this article!
