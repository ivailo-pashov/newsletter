If you ran the tests at the service level, contract tests for other services would, of course, fail. How to define contracts at the service level using OpenAPI and Spring Cloud Contract is explained in this post.

Test Driven Development - Walkthrough and Best Practices

Test-driven development is a software development process that relies on the repetition of a very short development cycle: first the developer writes an (initially failing) automated test case, then produces the minimum amount of code to pass that test, and finally refactors the new code to acceptable standards.
