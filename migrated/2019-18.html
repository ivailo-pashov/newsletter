Service Virtualization Meets Java: Hoverfly Tutorial

In Microservices architectures you often need to test end-to-end how a service communicates with other services. Mocking is not a solution since it does not test the communication stack, and skips everything related to network protocol (i.e HTTP). Running the dependent service is also not an option because of the effort required to prepare the process each time.
Service virtualization is a technique used to simulate the behavior of dependencies of service by creating a proxy service, hence the test runs against a service (i.e. the full stack is tested) without booting up the real service.
