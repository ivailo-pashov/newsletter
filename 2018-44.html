How Java 10 Changes the Way We Use
Anonymous Inner Classes 

When a new feature is introduced to a programming language specification is introduced, subtle changes in the new, practical ways we can now write code from version to version go without much notice. 
Using var you can now use anonymous classes as a replacement of named classes. This is convenient if you want to encapsulate interactions with instances of this class and data attached to them inside a method.

The Evolution of Spring Fu 

The idea behind Spring Fu project is to go "fully functional" in both Kotlin and Java. The new APIs allow you configuring Spring Boot in a functional way, introduce Coroutines support, GraalVM native images support and various other features. This means a different way of thinking about bean definitions in Spring applications but also significant improvements in startup performance. 
Another great thing news is that Spring Fu has also been introduced in Spring Cloud Function 2.0. Check out how the new configuration approach in Spring Cloud Function helps reducing twice the cost when going serverless on AWS!
