Design Patterns for Microservices  

Microservice architecture has become the de facto choice for modern application development. Although it solves a lot of problems, it is not deprived of drawbacks. This brings the need to learn about these problems and solve them with reusable solutions. In this article you will learn the most commonly used design patterns in microservice architectures.

 
 
 
 

A Coroutines Framework for Java 

Inspired by coroutines in Go and Kotlin, this framework provides the missing support for coroutines in Java without using JNI or the need to perform bytecode manipulation from separate Java Agents. Check out this interesting tutorial on using coroutines focused on two main applications: declarations and executions.

Modeling Uncertainty with Reactive DDD 

If you want to build Reactive, distributed applications you must decide how to handle uncertainty regarding the delivery of messages, including multiple delivery and out-of-order delivery. Here you can see how Domain-Driven Design can aid with managing uncertainty through the use of good modelling.

Spring Boot Auto-Configuration Mystery Revealed 

Spring boot is magic, it is amazing how quickly you can set it setup. However, few of us really know what is happening under the hood when we run our application. In this article you can find out exactly what happens after you add your dependencies and start up the project.

How Much Memory is My Application Wasting? 

Even though memory scarcity is not much of an issue in these modern times, finding out how you can optimise your code can save a lot of resources and money.
